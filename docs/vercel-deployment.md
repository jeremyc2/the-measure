# Vercel Deployment

This app is prepared for Vercel as a public static site with:

- `vercel.json`, which sets the Bun runtime, `buildCommand`, `outputDirectory`, and SPA fallback rewrites.
- `public/index.html`, tracked in git as the stable document shell.
- `public/assets/`, generated by Tailwind and Bun's browser bundle.
- No serverless function, database, auth provider, or API route.

## Create The Project

1. Create or import the Git repository in Vercel.
2. Keep the build command as `bun run build`, matching `vercel.json`.
3. Keep the output directory as `public`.
4. Deploy a preview once so Vercel creates the project and shows the generated preview URL.

Useful docs:

- Vercel project config: https://vercel.com/docs/project-configuration/vercel-json
- Vercel builds: https://vercel.com/docs/deployments/builds

## Deploy

Push to the branch connected to Vercel, or deploy from the CLI:

```sh
bun run build
vercel deploy
vercel deploy --prod
```

After deployment, verify:

- `/` serves the static shell.
- `/essays` refreshes successfully through the rewrite.
- `/assets/index.js` and `/assets/styles.css` are served from the static output.
